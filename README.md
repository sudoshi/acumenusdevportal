## Refresh portal assets

```bash
/home/smudoshi/Github/dev.acumenus.net/scripts/sync-assets.sh
```

The refresh script copies Graphify views, the E2E screenshot catalogue, and curated reference docs from the local Parthenon checkout into the Apache document root with web-readable permissions. Graphify cache directories are excluded because they are not needed by the browser views.

The Reference tab is generated by:

```bash
/home/smudoshi/Github/dev.acumenus.net/scripts/sync-reference.sh
```

That script mirrors critical Parthenon docs, writes `/reference/manifest.json`, and regenerates operational snapshots under `/reference/generated/` for frontend routes, backend API route declarations, package versions, deploy script highlights, the `dev.acumenus.net` Apache vhost, and Graphify/screenshot evidence counts.

The Database tab is generated by SchemaSpy plus PostgreSQL catalog metadata:

```bash
SCHEMASPY_DB_PASSWORD=... /home/smudoshi/Github/dev.acumenus.net/scripts/generate-schemaspy.sh
```

Defaults target `parthenon` on `127.0.0.1:5432` as `smudoshi` and publish separate reports for `app`, `omop`, and `vocab`. The generator downloads SchemaSpy and the PostgreSQL JDBC driver into `tools/schemaspy/` if needed, runs SchemaSpy with `-norows`, writes `/schemaspy/{app,omop,vocab}/`, and builds `/schemaspy/manifest.json`, `/schemaspy/index.html`, `/schemaspy/summary.md`, and per-schema `analysis.html` and `metadata.json` files. The password is read from `SCHEMASPY_DB_PASSWORD` or `PGPASSWORD` and is not written into generated portal files.

## Smoke check

```bash
/home/smudoshi/Github/dev.acumenus.net/scripts/smoke-portal.sh
```

The smoke script normalizes public file permissions, validates the Graphify, screenshot, reference, SchemaSpy, and `build-info.json` manifests, and checks that the local HTTPS vhost redirects unauthenticated requests into Authentik SSO. Set `DEV_PORTAL_REQUIRE_SSO=0` to run the legacy direct-content checks. It uses `DEV_PORTAL_HOST` and `DEV_PORTAL_IP` overrides when the vhost should be checked against a different hostname or loopback target.

## Build info

```bash
/home/smudoshi/Github/dev.acumenus.net/scripts/generate-build-info.sh
```

Writes `public/build-info.json` with the portal repo's git SHA / branch / dirty flag, the upstream Parthenon SHA, the host that built it, and an ISO timestamp. The portal footer reads this file to render the build stamp; if absent, it falls back to the newest manifest timestamp. `sync-assets.sh` calls this script at the end of every refresh, so the build stamp updates whenever assets are re-synced.

## Periodic refresh (systemd timer)

To keep the "What's new since last visit" banner and freshness badges meaningful, the asset sync should run on a schedule. Unit files live in `systemd/`:

```bash
/home/smudoshi/Github/dev.acumenus.net/systemd/install.sh
```

The installer copies `dev-portal-refresh.service` and `dev-portal-refresh.timer` into `/etc/systemd/system`, reloads systemd, and enables the timer. Default cadence: hourly at `:07` (with a 2-minute jitter), plus once 2 minutes after boot. Edit `dev-portal-refresh.timer` and re-run the installer to change the cadence.

Inspect the timer:

```bash
sudo -A systemctl status dev-portal-refresh.timer
sudo -A systemctl list-timers dev-portal-refresh.timer
journalctl -u dev-portal-refresh.service --since '2 hours ago'
```

Run a one-shot refresh on demand without waiting for the timer:

```bash
sudo -A systemctl start dev-portal-refresh.service
```

## SSO

`dev.acumenus.net` is protected by Authentik through Apache `mod_auth_openidc`.
The Authentik application slug is `dev-portal`, with callback
`https://dev.acumenus.net/oidc/callback`. The Apache SSL vhost includes the
secret-bearing local file:

- `/etc/apache2/dev-portal-oidc/dev.acumenus.net.conf`

That include is intentionally outside the repository. It contains the OIDC
client secret and `OIDCCryptoPassphrase`, so keep it root-readable only.

## TLS

`dev.acumenus.net` uses a Let's Encrypt certificate installed by Certbot's Apache plugin. The live files are:

- `/etc/apache2/sites-available/dev.acumenus.net.conf`
- `/etc/apache2/sites-available/dev.acumenus.net-le-ssl.conf`
- `/etc/letsencrypt/live/dev.acumenus.net/fullchain.pem`
- `/etc/letsencrypt/live/dev.acumenus.net/privkey.pem`
